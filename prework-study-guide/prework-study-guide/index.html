Git

  • a repository or repo contains all your projects files and each files revision history
  • Git commit a snapshot of your repo at a certain point in its timeline
  • When workin in the repo, you make new branches to make changes, this way if you need to revert these changes, or your code does not work, you can easily go back to a previous version
  • Git work flow -
    1. Create a Branch
    2. Make changes
    3. create a pull request, this allows collaborators to review and give feedback
    4. Address review comments on GitHub
    5. Merge the pull request into the main branch
    6. delete the branch
  • use cd and the file name to navigate to files in the git command window
  • Use pwd to display current location and file path
  • Use code . to open the VS code window
  • Use git status to check what branch you are on
  • git checkout -b branch-name: creates a new branch and switches to it
  • The checkout command is to move the working branch to a new branch
  • Use git add -A to add all changes to the current working branch
  • Use git commit to commit these files
  • git commit -m "notes" to add a note to your commit
  • git pull origin main to receive a branches modifications into the local environment.
  • After syncing with the remote branch, use use git push origin to push your changes to it
  • Use the clear function to give you a blank window
  • git diff to see the modification you have made
  • You can drag files into notepad to edit them and then use git to save and upload these changes
  • To delete a branch in git, use git branch -d branch name
