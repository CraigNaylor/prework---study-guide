Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The code in the Head Element contains information or metadata that helps the browser render the web page
  • The body element represents the visible content shown to the user.
  • All web page elements that we want the user to see and interact with are in the body element
  • When creating ordered or unordered lists, all lis items must be wrapped in the li element
  • Headers must always be in order, i.e. h1, h2, h3 etc.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are 3 ways to style a webpage in CSS
    1. Inline CSS
    2. Internal CSS
    3. External CSS style sheet
  • Inline is useful for quick & permanent chnages but is less flexible
  • External style sheets is the most popular, it allows developers to keep all CSS rules in a seperate file, making design changes easier
  • The extrnal style sheet can be added using the link function
  • Selectors are used to define elements or attributes that the declaration applies to
  • Declarations contain the CSS property we want to apply and the value of the property I.E. color:blue
  • The star symbol can be used as a wildcard, this will apply a rule to all elements visible on the page
  • Always Use DRY code - Do Not Repeat Yourself
  • Using a period before a selector I.E. - .card designates it as a class selector. Any HTML element assigned to the card class will gain the class assignemnts from CSS

Git

  • a repository or repo contains all your projects files and each files revision history
  • Git commit a snapshot of your repo at a certain point in its timeline
  • When workin in the repo, you make new branches to make changes, this way if you need to revert these changes, or your code does not work, you can easily go back to a previous version
  • Git work flow -
    1. Create a Branch
    2. Make changes
    3. create a pull request, this allows collaborators to review and give feedback
    4. Address review comments on GitHub
    5. Merge the pull request into the main branch
    6. delete the branch
  • use cd and the file name to navigate to files in the git command window
  • Use pwd to display current location and file path
  • Use code . to open the VS code window
  • Use git status to check what branch you are on
  • git checkout -b branch-name: creates a new branch and switches to it
  • The checkout command is to move the working branch to a new branch
  • Use git add -A to add all changes to the current working branch
  • Use git commit to commit these files
  • git commit -m "notes" to add a note to your commit
  • git pull origin main to receive a branches modifications into the local environment.
  • After syncing with the remote branch, use use git push origin to push your changes to it
  • Use the clear function to give you a blank window
  • git diff to see the modification you have made
  • You can drag files into notepad to edit them and then use git to save and upload these changes
  • To delete a branch in git, use git branch -d branch name
